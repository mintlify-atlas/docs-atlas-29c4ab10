---
title: "Get Project"
description: "Retrieve a single project by ID, including its rooms and progress history."
---

## Overview

Returns a project and all of its associated rooms. If the project has an `overall_progress` greater than `0` but no recorded `progress_history` entries, the server automatically seeds an initial history entry at the project's creation date and a current entry at the time of the request.

`GET /api/projects/{id}`

## Path parameters

<ParamField path="id" type="string" required>
  The project's unique identifier (MongoDB ObjectId string). Obtained from the `id` field returned by [List Projects](/api/projects/list) or [Create Project](/api/projects/create).
</ParamField>

## Response

<ResponseField name="id" type="string" required>
  Unique identifier for the project.
</ResponseField>

<ResponseField name="name" type="string" required>
  Display name of the project.
</ResponseField>

<ResponseField name="reference_type" type="string" required>
  Spatial reference method for zones. One of `images` or `3d_model`.
</ResponseField>

<ResponseField name="model_url" type="string | null">
  Server-relative path to the 3D model file, or `null`.
</ResponseField>

<ResponseField name="target_completion_date" type="string | null">
  ISO 8601 date string for the scheduled completion date, or `null`.
</ResponseField>

<ResponseField name="overall_progress" type="number" required>
  Aggregate construction progress across all zones (0–100).
</ResponseField>

<ResponseField name="budget" type="number" required>
  Total project budget.
</ResponseField>

<ResponseField name="progress_history" type="object[]" required>
  Ordered list of progress snapshots. May be seeded automatically on first retrieval if the project has progress but no history.

  <Expandable title="properties">
    <ResponseField name="date" type="string" required>
      ISO 8601 timestamp of when this snapshot was recorded.
    </ResponseField>

    <ResponseField name="progress" type="number" required>
      Overall progress value at the time of this snapshot (0–100).
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="rooms" type="object[]" required>
  Array of zone/room objects belonging to this project.

  <Expandable title="properties">
    <ResponseField name="id" type="string" required>
      Unique identifier for the room.
    </ResponseField>

    <ResponseField name="project_id" type="string" required>
      ID of the parent project.
    </ResponseField>

    <ResponseField name="name" type="string" required>
      Display name of the room or zone.
    </ResponseField>

    <ResponseField name="reference_image_url" type="string | null">
      URL or path to the reference photograph for this zone.
    </ResponseField>

    <ResponseField name="model_camera_position" type="object | null">
      Saved camera position within the 3D model for this zone. Shape depends on the viewer implementation.
    </ResponseField>

    <ResponseField name="mesh_assignments" type="string[]">
      UUIDs of 3D model meshes assigned to this room.
    </ResponseField>

    <ResponseField name="current_percent" type="number" required>
      Most recent AI-estimated completion percentage for this zone (0–100).
    </ResponseField>

    <ResponseField name="created_at" type="string" required>
      ISO 8601 timestamp for when the room was created.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="created_at" type="string" required>
  ISO 8601 timestamp for when the project was created.
</ResponseField>

## Examples

<CodeGroup>

```bash cURL
curl https://your-domain.com/api/projects/6627a1b2c3d4e5f6a7b8c9d0
```

```typescript TypeScript
const id = '6627a1b2c3d4e5f6a7b8c9d0';

const response = await fetch(`https://your-domain.com/api/projects/${id}`);

if (!response.ok) {
  const error = await response.json();
  throw new Error(error.error);
}

const project = await response.json();
console.log(project.rooms.length); // number of zones
```

</CodeGroup>

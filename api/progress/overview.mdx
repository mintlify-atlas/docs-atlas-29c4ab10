---
title: "Progress API"
description: "List and create daily progress check-in entries for construction zones."
---

Progress entries record a point-in-time snapshot of a zone's completion state. Each entry stores a photo URL alongside optional AI analysis results. When you create an entry you can include AI output generated by the [Analyze endpoint](/api/analyze), or leave the AI fields blank and fill them in later.

<Note>
Progress entries are stored in Supabase (PostgreSQL), while zones are stored in MongoDB. The `room_id` field links entries to their parent zone by the zone's MongoDB ObjectId string.
</Note>

---

## List progress entries

<br />

`GET /api/progress`

Returns progress entries ordered by `captured_at` descending (newest first). Pass `room_id` to scope the results to a single zone.

### Query parameters

<ParamField query="room_id" type="string">
  Zone identifier. When omitted, entries from all zones are returned.
</ParamField>

### Response

Returns an array of progress entry objects.

<ResponseField name="id" type="string">
  Unique entry identifier (UUID).
</ResponseField>

<ResponseField name="room_id" type="string">
  Parent zone identifier.
</ResponseField>

<ResponseField name="photo_url" type="string">
  Public URL of the progress photo in Supabase Storage.
</ResponseField>

<ResponseField name="ai_estimated_percent" type="number | null">
  Completion percentage estimated by AI analysis (0–100), or `null` if not analysed.
</ResponseField>

<ResponseField name="ai_confidence" type="string | null">
  AI confidence rating: `"high"`, `"medium"`, or `"low"`. `null` if not analysed.
</ResponseField>

<ResponseField name="visible_completed" type="string[] | null">
  Array of work items the AI identified as visibly complete in the photo.
</ResponseField>

<ResponseField name="still_missing" type="string[] | null">
  Array of work items the AI identified as not yet complete.
</ResponseField>

<ResponseField name="notes" type="string | null">
  Brief free-text observation from the AI or a human reviewer.
</ResponseField>

<ResponseField name="captured_at" type="string">
  ISO 8601 timestamp when the photo was taken / entry was created.
</ResponseField>

<CodeGroup>

```bash cURL
curl 'https://your-buildpulse-app.vercel.app/api/progress?room_id=64f1a2b3c4d5e6f7a8b9c0d2'
```

```typescript TypeScript
const res = await fetch('/api/progress?room_id=64f1a2b3c4d5e6f7a8b9c0d2')
const entries = await res.json()
// [{ id: 'uuid-...', ai_estimated_percent: 67, ai_confidence: 'high', ... }, ...]
```

</CodeGroup>

## Create a progress entry

<br />

`POST /api/progress`

Creates a new progress entry and returns it with status `201`.

A typical workflow is:

1. Upload the photo via [`POST /api/upload`](/api/upload) to get a `photo_url`.
2. Run AI analysis via [`POST /api/analyze`](/api/analyze) to get completion data.
3. Call this endpoint with the photo URL and analysis results.

### Body parameters

<ParamField body="room_id" type="string" required>
  Identifier of the zone this entry belongs to.
</ParamField>

<ParamField body="photo_url" type="string" required>
  Public URL of the progress photo. Upload the file first using `POST /api/upload`.
</ParamField>

<ParamField body="ai_estimated_percent" type="number">
  Completion percentage from AI analysis (0–100). Defaults to `null`.
</ParamField>

<ParamField body="ai_confidence" type="string">
  Confidence level from AI analysis: `"high"`, `"medium"`, or `"low"`. Defaults to `null`.
</ParamField>

<ParamField body="visible_completed" type="string[]">
  Work items identified as complete. Defaults to `null`.
</ParamField>

<ParamField body="still_missing" type="string[]">
  Work items not yet complete. Defaults to `null`.
</ParamField>

<ParamField body="notes" type="string">
  Free-text notes. Defaults to `null`.
</ParamField>

### Response

Returns the created progress entry with HTTP `201`.

<CodeGroup>

```bash cURL
curl -X POST 'https://your-buildpulse-app.vercel.app/api/progress' \
  -H 'Content-Type: application/json' \
  -d '{
    "room_id": "64f1a2b3c4d5e6f7a8b9c0d2",
    "photo_url": "https://xyz.supabase.co/storage/v1/object/public/progress-photos/1700000000-kitchen.jpg",
    "ai_estimated_percent": 67,
    "ai_confidence": "high",
    "visible_completed": ["framing complete", "drywall installed", "rough electrical done"],
    "still_missing": ["flooring", "cabinet installation", "paint"],
    "notes": "Good progress since last check-in. Flooring delivery expected next week."
  }'
```

```typescript TypeScript
// 1. Upload photo
const uploadRes = await fetch('/api/upload', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    dataUrl: 'data:image/jpeg;base64,/9j/4AAQ...',
    fileName: 'kitchen-checkin.jpg',
    bucket: 'PROGRESS_PHOTOS',
  }),
})
const { url: photoUrl } = await uploadRes.json()

// 2. Run AI analysis
const analyzeRes = await fetch('/api/analyze', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    progressPhoto: 'data:image/jpeg;base64,/9j/4AAQ...',
    roomName: 'Kitchen',
  }),
})
const analysis = await analyzeRes.json()

// 3. Save progress entry
const res = await fetch('/api/progress', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    room_id: '64f1a2b3c4d5e6f7a8b9c0d2',
    photo_url: photoUrl,
    ai_estimated_percent: analysis.percent_complete,
    ai_confidence: analysis.confidence,
    visible_completed: analysis.visible_completed,
    still_missing: analysis.still_missing,
    notes: analysis.notes,
  }),
})
const entry = await res.json()
// { id: 'uuid-...', room_id: '...', ai_estimated_percent: 67, ... }
```

</CodeGroup>

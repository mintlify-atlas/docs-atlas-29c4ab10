## Prerequisites

Before you begin, make sure you have accounts and credentials for each service BuildPulse depends on:

- **Node.js 18+** — [nodejs.org](https://nodejs.org)
- **MongoDB Atlas account** — for project data and GridFS model storage
- **Google AI Studio API key** — for Gemini 2.0 Flash vision analysis
- **Auth0 application** — for authentication (all routes are protected by default)
- **Supabase project** — for relational data and file storage buckets
- **OpenNote API key** — for AI-generated video stakeholder reports

<Steps>

<Step title="Clone the repository">

```bash
git clone https://github.com/pranad-dev/buildpulse.git
cd buildpulse
```

</Step>

<Step title="Install dependencies">

```bash
npm install
```

</Step>

<Step title="Configure environment variables">

Copy the example env file and fill in your credentials:

```bash
cp .env.example .env.local
```

Open `.env.local` and replace each placeholder with your actual values. See the [Environment Variables](/environment-variables) page for a full description of every key.

<Warning>
  Never commit `.env.local` to git. It contains secrets and is already listed in `.gitignore`.
</Warning>

</Step>

<Step title="Set up Supabase">

BuildPulse uses Supabase for relational data (projects, zones, progress entries) and file storage.

**Create the database schema**

1. Open your [Supabase dashboard](https://supabase.com/dashboard)
2. Click **SQL Editor** in the left sidebar, then **New query**
3. Paste the full schema below and click **Run**

```sql
-- Enable required extensions
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";

-- Drop existing tables if re-running
DROP TABLE IF EXISTS progress_entries CASCADE;
DROP TABLE IF EXISTS rooms CASCADE;
DROP TABLE IF EXISTS projects CASCADE;

-- Projects table
CREATE TABLE projects (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    name TEXT NOT NULL,
    reference_type TEXT NOT NULL CHECK (reference_type IN ('images', '3d_model')),
    model_url TEXT,
    target_completion_date DATE,
    overall_progress INTEGER DEFAULT 0 CHECK (overall_progress >= 0 AND overall_progress <= 100),
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Rooms (zones) table
CREATE TABLE rooms (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    project_id UUID NOT NULL REFERENCES projects(id) ON DELETE CASCADE,
    name TEXT NOT NULL,
    reference_image_url TEXT,
    model_camera_position JSONB,
    current_percent INTEGER DEFAULT 0 CHECK (current_percent >= 0 AND current_percent <= 100),
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Progress entries table
CREATE TABLE progress_entries (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    room_id UUID NOT NULL REFERENCES rooms(id) ON DELETE CASCADE,
    photo_url TEXT NOT NULL,
    ai_estimated_percent INTEGER CHECK (ai_estimated_percent >= 0 AND ai_estimated_percent <= 100),
    ai_confidence TEXT CHECK (ai_confidence IN ('high', 'medium', 'low')),
    visible_completed TEXT[],
    still_missing TEXT[],
    notes TEXT,
    captured_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Indexes for performance
CREATE INDEX idx_rooms_project_id ON rooms(project_id);
CREATE INDEX idx_progress_entries_room_id ON progress_entries(room_id);
CREATE INDEX idx_projects_created_at ON projects(created_at DESC);
CREATE INDEX idx_progress_entries_captured_at ON progress_entries(captured_at DESC);
```

You should see "Success. No rows returned" — that means the schema was created correctly.

**Create storage buckets**

BuildPulse needs three public storage buckets in Supabase Storage:

1. Click **Storage** in the left sidebar, then **New bucket**
2. Create each of the following buckets with **Public bucket** toggled **on**:

| Bucket name | Contents |
|---|---|
| `models` | `.glb` and `.gltf` 3D model files |
| `reference-images` | Zone reference photos |
| `progress-photos` | Daily check-in site photos |

<Tip>
  Bucket names must match exactly. The application references them as `models`, `reference-images`, and `progress-photos`.
</Tip>

**Get your Supabase credentials**

1. Go to **Project Settings** → **API**
2. Copy your **Project URL** and **anon/public key**
3. Add them to `.env.local`:

```bash
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

</Step>

<Step title="Start the development server">

```bash
npm run dev
```

</Step>

<Step title="Open the app">

Navigate to [http://localhost:3000](http://localhost:3000).

Auth0 middleware protects all routes by default (see `middleware.ts`). You will be redirected to the Auth0 login page if you are not authenticated. After logging in, you'll be returned to the dashboard.

<Note>
  The middleware matcher covers all routes except Next.js static assets (`_next/static`, `_next/image`, `favicon.ico`). There is no public unauthenticated path in this app.
</Note>

</Step>

<Step title="Create your first project">

1. Click **New Project** on the dashboard
2. Enter a project name and target completion date
3. Upload a `.glb` or `.gltf` 3D model (up to 50MB) — it will be stored via MongoDB GridFS
4. Click **Create Project**

You'll be redirected to the project dashboard where you can assign zones and upload progress photos.

</Step>

</Steps>
